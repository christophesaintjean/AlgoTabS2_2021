Introduction¶

Définition¶

On dfinit le problme du tri comme:

Entre: une squence de valeurs $a_1, a_2, ..., a_n$ comparables

Sortie: une permutation $\sigma$ (un rarrangement) telle que la squence $$a_{\sigma(1)}, a_{\sigma(2)}, ..., a_{ \sigma(n)}$$ soit ordonne (croissant ou dcroissant)

Clé de tri¶

On parle de cl de tri pour dsigner l'lment sur lequel l'ordre est mis en place.

  • Type de cl usuelles: nombre, chaine, liste
  • Cl dfinie par l'utilisateur. Ex. pour une chaine
    • ordre lexicographique (par dfaut)
    • nombre de caractres pour une chaine
    • nombre de consonnes puis nombre de voyelles
    • mois de l'anne: 'Janvier' < 'Fvrier'

  • Cl simple dans une structure plus complexe.

    Ex.: Tuple (JJ, MM, YYYY)

Le langage de programmation fait donc des choix de cls par dfaut.

Il revient au programmeur de les adapter si besoin est, et de dfinir sa propre de comparaison (voir Bonus).

Classification des algorithmes de tri¶

Tri en place¶

Un algorithme de tri est dit en place lorsque jamais plus d'un nombre constant d'lments est stock hors du tableau quelque soit sa taille.

Tri stable¶

Un algorithme de tri est dit stable lorsque l'ordre des lments ayant une cl identique est maintenu.

Exemple:

  • cl de tri : nombre de caractres pour une chaine
  • T = ['cb', 'e', 'hi', 'A'] -> Tri stable -> ['e', 'A', 'cb', 'hi']
  • T = ['cb', 'e', 'hi', 'A'] -> Tri instable -> ['A', 'e', 'cb', 'hi']

On peut rendre stable un algorithme de tri en introduisant une cl secondaire.

Tri par comparaison ou non¶

  • Un algorithme de tri peut utiliser des comparaisons "<"
  • ou non... (voir TP)

Tri incrémental¶

Un algorithme de tri est incrmental (ou en ligne) si il est capable de traiter les donnes une par une sans disposer du tableau en entier.

Pour nous, le tri sera incrmental si l'arrive d'une donne ne force pas tout retrier.

Tri par sélection¶

L'algorithme¶

  • Donnes : T un tableau $n$ valeurs comparables
  • Rsultat: T est tri par ordre croissant
  • Principe: Pour un i allant de 1 n-1 par ordre croissant:
    • Chercher le minimum de T[i..n] -> imin
    • Echanger T[imin] et T[i]

Caractéristiques: Tri par sélection¶

En place ? etc ?

Il est:

  • en place
  • stable
  • par comparaison
  • non incrmental

Analyse du Tri par sélection¶

  • Terminaison: OK
  • Correction de l'algorithme par invariant
  • Complexit (asymptotique) de l'algorithme.

Preuve de correction par rcurrence sur i:

  • Pour i = 1, l'algorithme recherche le plus petit lment de T et le met en premire position. => Le sous-tableau T[1..1] est tri

  • Supposons T[1..i-1] tri.

    L'algorithme recherche le plus petit lment de T[i..n] et le met en position i.

    • T[i] <= tous les lments de T[i..n] (car min).
    • T[i] >= tous les lments de T[1..i-1] car sinon il aurait < qu'un des lments de T[1..i-1] (impossible).

  • On en conclut que T[1..i] est tri la fin de chaque boucle sur i.

  • A la fin, i = n-1 donc T[1..n-1] est tri et comme T[n] >= tous les lments de T[1..n-1], CQFD.

Efficience de l'algorithme

Nombre d'changes ?

Pour i fix, 1 change.

Nombre total d'changes: n-1

Nombre de comparaisons ?

Pour i fix, n-i comparaisons.

  • i = 1 -> n - 1 comparaisons
  • i = 2 -> n - 2 comparaisons
  • ...
  • i = n - 2 -> 2 comparaisons
  • i = n - 1 -> 1 comparaison

Nombre total de comparaisons: $$1+2+3+...+(n-2)+(n-1) = \frac{n*(n-1)}{2} \sim n^2$$

def tri_selection(T):
    n = len(T)
    for i in range(0, n-1):
        imin = i
        for j in range(i+1, n):
            if T[j] < T[imin]:
                imin = j
        T[i], T[imin] = T[imin], T[i]
    return T

Tri par insertion¶

L'algorithme¶

  • Donnes : T un tableau $n$ valeurs comparables
  • Rsultat: T est tri par ordre croissant

  • Principe: Pour i allant de 2 n par ordre croissant:

    Inserr T[i] dans le tableau T[1..i-1] tri en dcalant vers la droite toutes les valeurs qui lui sont >

source: Tri par insertion (Wikipdia)

Tri par insertion¶

Caractéristiques du Tri par insertion¶

Il est:

  • en place
  • stable (car inf. strict)
  • par comparaison
  • incrmental

Analyse du Tri par insertion¶

  • Terminaison de l'algorithme: OK
  • Correction de l'algorithme par invariant:
    • T[1] Tri
    • T[1..i-1] est tri -> aprs -> T[1..i] tri
    • Donc T[1..n] tri la fin.

Efficience de l'algorithme

Le nombre de comparaisons est proportionnel au nombre d'changes.

  • Pour i fix, on distingue plusieurs cas, cl = T[i]:

    • 2 comparaisons si T[i-1] $\leq$ cl
    • 2*(i-1) comparaisons si cl $<$ T[j] pour tout j=1..i-1.
    • i (= 2*(i/2)) comparaisons en moyenne

Nombre total de comparaisons:

  • Cas favorable:
$$nbComp(n) \geq \sum_{i=2}^n 2 = 2 *\sum_{i=1}^{n-1} 1 =2(n-1) \sim n$$
  • Cas dfavorable
$$nbComp(n) \leq \sum_{i=2}^n 2*(i-1) = 2 * \sum_{i=1}^{n-1} i = n * (n-1) \sim n^2$$

def tri_insertion(T):
    n = len(T)
    for i in range(1, n):
        cle = T[i]
        j = i - 1
        while j >= 0 and cle < T[j]:
            T[j+1] = T[j]
            j = j - 1
        T[j+1] = cle 
    return T
